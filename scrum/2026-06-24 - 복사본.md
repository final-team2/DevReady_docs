## 오늘 한 일

### 1. 면접 진행 화면 구성 정리
- 면접 진행 화면에서 질문 진행률, 현재 질문, 영상/음성/텍스트 답변 영역 구조 확인
- 영상 분석 동의 여부에 따라 화면이 다르게 동작하도록 UX 방향 정리
- 영상 동의 사용자는 영상+음성 면접 화면을 기본으로 구성
- 영상 미동의 사용자는 음성만 답변을 기본으로 구성

### 2. 답변 방식 전환 UI 개선 방향 정리
- 기존 `답변 방식` 카드 영역 제거 방향으로 정리
- 답변 방식 전환 버튼을 `현재 질문` 카드 우측에 배치하기로 결정
- 버튼은 `카메라 켜기/끄기` 토글 + `텍스트로 답변` 중심으로 단순화
- 카메라를 끄면 음성만으로 답변한다는 안내 문구 추가 예정

### 3. 답변 방식 변경 로직 정리
- 답변 시작 전에는 답변 방식 변경 가능
- 답변 중에는 카메라/텍스트/음성 전환 불가
- 답변 완료 후에는 바로 전환하지 않고, `답변 다시 하기` 또는 다음 질문에서 변경 가능하도록 정리
- 다음 질문으로 넘어가면 다시 답변 방식 변경 가능하도록 흐름 정리

### 4. mock 면접 흐름 개선
- 5문항/10문항/15문항 선택값에 맞춰 mock 질문 개수를 반영하는 방향 정리
- 기본 질문 완료 후 mock 꼬리질문 흐름 추가 방향 정리
- 실제 AI 이력서 기반 질문 생성 API는 연결 통로만 있고, resumeId 기반 자동 연동은 아직 미완성인 것으로 확인

### 5. 면접 이탈 경고 정책 정리
- 면접 진행 중 새로고침, 뒤로가기, 상단 메뉴 이동, 탭 닫기 시 경고 필요
- 정상적으로 리포트 화면으로 이동하는 경우에는 경고 제외
- 답변 저장 불가 안내와 함께 이용권/멤버십 횟수 제한 관련 안내 문구 추가 예정
- 무제한 요금제와 횟수 제한 요금제가 모두 있으므로 확정형 차감 문구 대신 공통 안내 문구로 정리
